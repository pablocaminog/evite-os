Pages & Routes

Astro hybrid — static where possible, SSR where dynamic

Static Pages (pre-rendered)

/
Landing — hero, "Create Party" CTA, how it works

SSR Pages (dynamic)

/party/new
Create party form — title, date, location, description, image upload, organizer info
/manage/[token]
Organizer dashboard — party details, guest list + RSVP counts, add guests, send invites, export CSV
/invite/[rsvp_token]
Guest view — party info, cover image, RSVP form (name pre-filled, attending Y/N, count, dietary notes)
/magic-link
Enter email → receive link to management URL. Simple form, no flash of unauthed content.

API Routes (Astro endpoints)

POST /api/parties
Create party → returns management_token URL
POST /api/parties/[id]/guests
Add guest(s) to party (requires management_token in header)
POST /api/parties/[id]/invite
Send email/SMS invites to pending guests
POST /api/rsvp/[rsvp_token]
Submit or update RSVP — upserts status, count, notes
POST /api/magic-link
Request magic link by email → sends link if email matches a party
POST /api/upload
Upload cover image → stores in R2, returns key
GET /api/parties/[token]/export
Download guest list as CSV (management_token auth)
